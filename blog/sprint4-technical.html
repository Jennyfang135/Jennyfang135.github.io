Got blocked on a simple problem

What was the problem?

I was stuck with a challenge question in Repl.it Olympic event 2 - "Reverse a String" for hours, I had all sorts of problems with this challenge, small or big, using different problem solving skills, I had them sorted out eventually. The simple problem that I had with this challenge was I misuderstood the question. At the beginning I thought "String" in the question means only 1 word, like "Hello" or "world", when my original code failed the test, I realised that a String can be one word or mulitple words or sentences within the double quotes" ", like "Hello World!" or "I am going to school today."

What problem solving techniques did you use?

I solved this problem by reading the error messages, the error messages gave me the enough hints to figure out what wrong was my code.

How did you feel throughout the process?

I feel that debugging skills are so important and I need keeping on practicing them, to understand what the error messages mean, to put console.log in my code to test the output.

What did you learn?

- No rush to write code at the beginning
- Understanding what the problem is and how I am going to solve it.
- Reading the tests code first(if they are provided) and find out what kind of tests my code need to pass

Solved a problem in an elegant way

What was the problem?

The other problems I had with "Reverse a String" was how to reverse a String that has more than one word and space in between words? for example from "I am the master of the unjumble! Look at me go!" to "!og em ta kooL !elbmujnu eht fo retsam eht ma I"

What problem solving techniques did you use?

  1. Using Google search to find out if there is a method that can reverse a string? like string.reverse(), unfortuantely there is not; however there is a reverse method for reversing an array, and an string.split() method which converts a string to array. Bingo!
  2. write Pseudocode to plan how I am going to solve this problem:
    step 1- convert a string to an array by string.split();
    step 2 - reverse this array by method array.reverse() to create a reveresed array;
    step 3- reverse each element in the reversed array by repeat step 2 and 3;
    step 4- convert the every reversed element reversed array back to a String by method array.join()
  3. After all above steps done, I thought I successfully completed this challenge, but NO, I didn't. My test results failed!!! why? I was quite sure that my logic and the syntax of the code were right, so I used console.log to print out the outputs on the screen, but the output from console.log looks like I got the right answers, why I couldn't past the test?
  4. The problem solving skill I used next was to read the error messages carefully again , then I realised that there was an extra " " at the beginning of my reversed string! For example if I inupt "Hi! Jenny", my expecting output should be "ynneJ !iH", but the actual output of my code was " ynneJ !iH". It was an extra " " that failed my test! It was so hard to notice that there was an extra space there! Using a string.substring() method fixed this problem easily. Hooray, I past the challenge at last!

How did you feel throughout the process?

I feel debugging is so time consuming and a little small bug can take hours to figure it out and get it fixed, so how to improve my problem solving skills and debugging effeciency are something that I need to work more on them.

What did you learn?

- Analysing problems in a logic way
- breaking a big problem into small problems
- Writing pseudocodes
- Using googles to help find the useful methods
- Using console.log to print out the output
- Understanding error messages and reading them carefully
