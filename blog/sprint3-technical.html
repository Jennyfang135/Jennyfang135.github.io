An analogy to describe the differences between HTML and CSS

HTML is short for HyperText Markup Language, it is used to create the actual content of the webpage; "Markup language" means that, rather than using a programming language to perform functions, HTML uses tags to identify different types of content and the purposes they serve to the webpage, these tags have pretty intuitive names: Header tags, paragraph tags, body tags etc. CSS is short for Cascading Style Sheet, it is responsible for the design or style of the webstie, gives the HTML its styling and controls the layout and appearance of it, simply just make a webpage prettier, those slick colours, fancy fonts and background images, all thanks to CSS, CSS affects the entire mood and tones of a web page. You can think HTML as a builder while CSS is an artist. HTML and CSS have different code and syntex, however you can use CSS in an HTML file, but cannot use HTML in a CSS style sheet. As common practice, HTML and CSS are kept separate to ensure websites are built correctly before they're reformatted, and it also makes easier if you wish to change the appearances of your website in the future.

Describe what the DOM is and an example of how you might interact with it.

DOM stands for "Document Object Model" which is an application programming interface (API) for valid HTML and well-formed XML documents.
The DOM represents the document as nodes and objects. That way, programming languages can connect to the page.

DOM defines the logical structure of documents and the way a document is accessed and manipulated.
With the Document Object Model, programmers can build documents, navigate their structure, and add, modify, or delete elements and content. Anything found in an HTML document can be accessed, changed, deleted, or added using the Document Object Model.

For example, a Web page is a document. This document can be either displayed in the browser window or as the HTML source. But it is the same document in both cases. The Document Object Model (DOM) represents that same document so it can be manipulated. The DOM is an object-oriented representation of the web page, which can be modified with a scripting language such as JavaScript.

However, the DOM was designed to be independent of any particular programming language, making the structural representation of the document available from a single, consistent API. Though we focus exclusively on JavaScript but implementations of the DOM can be built for any language.

Put it simple, The backbone of an HTML document are tags.
According to DOM, every HTML-tag is an object. Nested tags are called “children” of the enclosing one.
The text inside a tag it is an object as well.
All these objects are accessible using JavaScript.

Explain the difference between accessing data from arrays and objects.

Array elements must be accessed by using the bracket notation.

Example of accessing an array item:
var data = myArray[index];

Example of looping over an array
for(var i=0; i < myArray.length; i++){
console.log(myArray[i]);
}

Data stored in simple objects, by contrast, is accessed by key, which you can think of as a name, An Object is logically a collection of properties. Each property is either a data property, or an accessor property. Object properties are accessed via their "key" names which are a string or symbol value Object properties can be accessed with 2 notations: dot & brackets

Example:
var myCat {
name: "mimi",
age: 3,
category: "scottish fold",
favouriteFood["fish", "egg", "milk"]
};

Above is an object called myCat, it contains 4 properties, name, age, category and faoriteFood which is reprsented by an array.

Access name property by dot
myCat.name; //return mimi

Access name property by bracket
myCat["name"]; //return mimi

Both way return the same value: mimi

Explain what functions are and why they are useful.

What is a function?

  • A function is a subprogram designed to perform a particular task.
  • Functions are executed when they are called. This is known as invoking a function.
  • Values can be passed into functions and used within the function.
  • Functions always return a value. In JavaScript, if no return value is specified, the function will return undefined.
  • Functions are objects.

Why functions are userful?

  • A function is a group of reuseable code which can be called anywhere in your program, no need to write code again and again
  • A big program can be divided into a number of small and manageable functions
  • It can be also useful for debugging because you can see the function’s name in a call stack
  • But some functions have no names, we call them anonymous functions, why are are anonymous functions useful?
    -If no name is needed because the function is only ever called in one place, then no point to add a name to whatever namespace you're in.
    -Anonymous functions are normally declared inline and inline functions have advantages in that they can access variables in the parent scopes.
